2、静态文件cdn

运行npm run cdn命令

该命令里面包含三个命令（npm run dist && node cdn && node bin/copy），需要发布cdn或者把开发环境迁移到测试以及生产环境都只需要一键执行npm run cdn即可。
下面是cdn命令的说明：

  • npm run dist命令
    • 去掉mock数据
    • 压缩css、js、图片等文件
    • 还原mock数据
  • node cdn命令
    • 把html和css里面的图片添加cdn,把requirejs添加cdn（包含所有的js和css）

  • node bin/copy命令

    • 把静态资源复制到cdn发布环境
    • 把html复制到对应的发布环境

    (注：1、开发环境应该和发布环境在同一目录。2、命令执行完后提交生产环境，联系毛坤发布cdn,23环境可以自由发布，如果云市场的html里面图片路径变化后需要修改ftl模板 )

3、版本控制

因为项目中使用了requirejs来对路径统一管理，以及模块化，通过对比r.js、fis3、gulp等构建工具后发现没有一个单独的构建工具能给出一个完整的解决方案。通过研究发现r.j+fis3能完美解决，但是考虑到时间的紧迫性，所以最终得出了一个不是最优但是最简单的方案。

解决方案

  • 通过第一次发布cdn后，若以后某一个js、css文件变化，我们在发布时需要修改该文件名，文件命名方式是文件名+ ‘_’ + 版本号，同步更新require.config.js里面的路径。
  • 通过第一次发布cdn后，若图修改后直接修改图片名称来进行版本化
